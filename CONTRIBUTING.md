# Guia de Contribuição

Obrigado por considerar contribuir para o projeto Azure Technical Translation! Este documento fornece diretrizes e instruções para contribuições.

## Código de Conduta

Todos os contribuidores devem aderir ao nosso Código de Conduta de Comunidade. Comportamento respectoso e profissional é esperado de todos.

## Como Contribuir

### Reportar Bugs

Antes de criar relatórios de bugs, verifique se o problema já foi reportado. Quando você estiver criando um relatório de bug, inclua o máximo de detalhes possível:

- **Use um título descritivo**
- **Descreva o comportamento exato observado**
- **Descreva o comportamento esperado**
- **Inclua exemplos de código específicos**
- **Inclua screenshots se possível**
- **Indique a versão do Python e dependências**

### Sugerir Melhorias

Sugestões de melhorias são sempre bem-vindas! Ao criar uma sugestão, inclua:

- **Use um título descritivo**
- **Forneça descrição em detalhes da melhoria sugerida**
- **Listar exemplos de implementação se possível**
- **Explicar por que essa melhoria seria útil**

### Pull Requests

- Siga os guias de estilo Python (PEP 8)
- Inclua testes apropriados
- Atualize documentação conforme necessário
- Mantenha histórico de commit limpo

## Processo de Desenvolvimento

1. Fork o repositório
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Commit suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

## Padrões de Código

- Use Black para formatação
- Use mypy para verificação de tipos
- Use flake8 para linting
- Mantenha cobertura de testes acima de 80%

## Documentação

- Escreva docstrings em formato Google
- Atualize README quando necessário
- Mantenha comentários de código claros e úteis

## Licença
